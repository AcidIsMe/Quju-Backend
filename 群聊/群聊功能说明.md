# 群聊功能实现说明

## 设计概述

基于已有 IM 系统的扩展，利用 `im_messages` 表的 `entity_type` 和 `entity_id` 字段区分私聊与群聊。

- **群聊 entity_type**: `"group"`
- **群聊 entity_id**: `"team:{teamId}"`（以小队ID作为群聊标识）
- **成员验证**: 复用 `team_members` 表
- **已读追踪**: 新增 `group_chat_read_markers` 表

## 数据流

### 发送消息

用户 (WebSocket) → `ImWebSocketServer.handleSendMessage()`
1. 校验 `entity_type = "group"` 时，查 `team_members` 验证发送方是小队成员
2. 写入 `im_messages` 表
3. 查 `team_members` 获取所有成员列表
4. 向所有在线的群成员广播 `new_message`

### 获取会话列表

客户端 → `GET /api/im/conversations`
1. 查 `im_messages` 获取用户参与的私聊，按 `entity_id` 分组取最新
2. 查 `team_members` 获取用户加入的所有小队
3. 对每个小队查 `im_messages` 取最新一条消息
4. 合并私聊和群聊，按最后消息时间倒序

### 已读标记

WebSocket `mark_read` + REST `POST /api/im/messages/read`

- **群聊**: 更新 `group_chat_read_markers` 表中 `last_read_at`（用户最后阅读时间）
- **私聊**: 逐条标记 `im_messages.read_at`（不变）

### 未读数统计

- **群聊**: `count(im_messages where created_at > last_read_at AND sender_id != me)`
- **私聊**: `count(im_messages where read_at IS NULL AND sender_id != me)`

## WebSocket 消息格式

### 发送群聊消息

```json
{
  "type": "send_message",
  "entity_type": "group",
  "entity_id": "team:{teamId}",
  "content": "大家好！",
  "msg_type": "text"
}
```

### 收到群聊消息推送

```json
{
  "type": "new_message",
  "message_id": "uuid",
  "entity_type": "group",
  "entity_id": "team:{teamId}",
  "sender_id": "uuid",
  "content": "大家好！",
  "created_at": "2024-06-15T10:00:00"
}
```

### 群聊输入状态

```json
{
  "type": "typing",
  "entity_type": "group",
  "entity_id": "team:{teamId}",
  "user_id": "uuid"
}
```

### 群聊已读标记

```json
{
  "type": "mark_read",
  "entity_type": "group",
  "entity_id": "team:{teamId}"
}
```

## REST 接口

所有 IM REST 接口（`POST /api/im/messages`、`GET /api/im/conversations` 等）已自动支持群聊，只需传入 `entity_type=group` 和 `entity_id=team:{teamId}`。

## 注意事项

1. 群聊成员资格来源于 `team_members` 表，加入/退出小队自动同步到群聊
2. 消息撤回仍遵循 2 分钟限制
3. 群聊未读数基于 `last_read_at` 时间戳计算，精度到秒级
4. WebSocket 当前为单设备模式，多设备支持需扩展 `SessionManager`
